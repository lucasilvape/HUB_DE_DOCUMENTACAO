    📊 FALCONI_PYTHON¶

    Sistema Automatizado de Geração de Indicadores e KPIs para Análise de Seguros¶

    Projeto de Business Intelligence (BI) que automatiza a geração de 7 indicadores/KPIs críticos de seguros através de pipelines Python orquestrados, com integração a SQL Server e persistência em arquivos Excel.

    🎯 Visão Geral e Arquitetura¶

    Objetivo do Projeto¶

    O FALCONI_PYTHON é um sistema de Business Intelligence que automatiza a geração de indicadores e KPIs críticos para a análise de performance em seguros. O projeto:

    • ✅ Orquestra a execução de múltiplos indicadores em lote ou individualmente
    • ✅ Integra-se a SQL Server para extração de dados de seguros
    • ✅ Aplica fórmulas de cálculo específicas para cada indicador (frequência, custo médio, recuperação, etc)
    • ✅ Persiste resultados em arquivos Excel estruturados com timestamps
    • ✅ Segue padrão de herança para garantir consistência e reutilização de código

    Indicadores Gerados¶

    ID Nome Objetivo Fórmula Principal
    11 Cross Sell Bancassurance Medir adoção de cross-sell de produtos de vida COUNT(clientes com 2+ produtos) / COUNT(total clientes) × 100
    131 Índice de Renovação RCO Identificar renovações de apólices RCO IF(qde_novas_apolices > 0, 'SIM', 'NÃO')
    16 Frequência de Sinistros DM Taxa de sinistralidade em Danos Materiais (QDE_SINISTROS / QDE_APOLICES) × 100
    17 Recuperação Sinistros DM Taxa de ressarcimento por data de ocorrência (VENDA_SALVADOS + RECEB_RESSARCIMENTO) / PAGAMENTO_TOTAL
    17m Recuperação Sinistros DM (Mensal) Taxa de ressarcimento por data de movimento (VENDA_SALVADOS + RECEB_RESSARCIMENTO) / PAGAMENTO_TOTAL
    26 Prazo Médio Liquidação Dias médios até primeiro pagamento AVG(AGING_PAGAMENTO) [dias entre ocorrência e 1º pagamento]
    39 Custo Médio de Sinistros RCO Valor médio por sinistro indenizado SUM(VL_OPERACAO) / COUNT(pagamentos)

    Padrão de Arquitetura: Herança Query¶

    O projeto segue um padrão de design baseado em herança para máxima reutilização e consistência:

    # Classe Abstrata (ABC)
Query (classe base abstrata)
  ├── @abstractmethod query() -> tuple
  ├── abrir_conexao()
  ├── executar_query()
  └── imprimir_status()

# Queries Concretas (implementam a ABC)
├── QdeApolicesAtivas(Query)          # Conta apólices ativas por filtro
├── DimProdutoRamo(Query)             # Dimensão de produtos conforme ramo
├── MovimentacaoSinistros(Query)      # Extrai movimentos de sinistros
└── [outras queries específicas]

# Geradores de Indicador (usam as queries acima)
├── GerarID11(ambiente, data_execucao)       → ID11_Cross_Sell_Bancassurance
├── GerarID131(ambiente, data_execucao)      → ID131_Indice_de_Renovacao_RCO
├── GerarID16(ambiente, data_execucao)       → ID16_Indice_de_Frequencia_Sinistros
├── GerarID17(ambiente, data_execucao)       → ID17_Recuperacao_Sinistros
├── GerarID17m(ambiente, data_execucao)      → ID17m_Recuperacao_Sinistros_MENSAL
├── GerarID26(ambiente, data_execucao)       → ID26_Prazo_Medio_Liquidacao
└── GerarID39(ambiente, data_execucao)       → ID39_Custo_Medio_Sinistros_RCO

    🔧 Configuração e Autenticação¶

    Variáveis de Ambiente (.env)¶

    O projeto utiliza dotenv para armazenar credenciais de forma segura. Crie um arquivo .env na raiz do projeto com as seguintes variáveis:

    # .env (NÃO versionado - adicione à .gitignore)
USER_PROD=seu_usuario_sql_server
PASSWORD_PROD=sua_senha_sql_server

    ⚠️ IMPORTANTE: - O arquivo .env não deve ser versionado (já está em .gitignore) - As credenciais carregadas via dotenv são injetadas em autenticacao.py

    Conexão SQL Server¶

    Classe Principal: Conexoes (em conexao_banco.py)

    from conexao_banco import Conexoes

# Método 1: Criar conexão com nome específico
conexao = Conexoes(conexao='producao_i4')
conn = conexao.abrir_conexao()

# Método 2: Factory method (recomendado)
conexao = Conexoes.conexao_producao_i4()
conn = conexao.abrir_conexao()

    Estrutura de autenticacao.py¶

    import os
from dotenv import load_dotenv

load_dotenv()

autenticacao = {
    "producao_i4": {
        "server": os.getenv('USER_PROD'),      # Usuário SQL Server
        "password": os.getenv('PASSWORD_PROD') # Senha SQL Server
    }
}

    📊 Indicadores Detalhados¶

    📊 Expandir Lista Completa de 7 Indicadores

    ID11 - Cross Sell Bancassurance¶

    Objetivo: Medir a penetração de produtos de vida em seguros (cross-sell em ramos 23 e 28).

    Fórmula: 

    Percentual de Clientes com Cross-Sell = (Clientes com 2+ produtos / Total de Clientes) × 100

    Variáveis de Entrada (SQL): - DOCUMENTO_TOMADOR - CPF/CNPJ do cliente - CD_RAMO - Código do ramo de seguro (filtro: 23, 28) - ID_APOLICE - Identificador único da apólice - DH_EMISSAO - Data e hora de emissão

    Estrutura de Saída (Excel):

    Coluna Tipo Descrição
    DT_CORTE string Data de corte (formato: YYYYMMDD)
    CROSS_SELL string 'SIM' = cliente tem 2+ produtos, 'NÃO' = apenas 1 produto
    QDE_CLIENTES int Quantidade de clientes nesta categoria
    PER_CROSS float Percentual de clientes com cross-sell
    ID string '11' (ID do indicador)
    NM_INDICADOR string 'Cross Sell Bancassurance'

    Exemplo de Linha:

    DT_CORTE: 202504, CROSS_SELL: SIM, QDE_CLIENTES: 1250, PER_CROSS: 0.3500, ID: 11, NM_INDICADOR: Cross Sell Bancassurance

    Arquivo Gerado: 

    \\awsseg130006\ACTIO\ID - 11\ID 11 - Cross Sell Bancassurance - COMPETENCIA 202504 - gerado em 15-11-2024 h13 m41 s54.xlsx

    ID131 - Índice de Renovação RCO¶

    Objetivo: Medir a taxa de renovação de apólices em Responsabilidade Civil Operacional (RCO).

    Fórmula: 

    Índice de Renovação = (Apólices Renovadas / Apólices com Vigência Finalizada) × 100

    Variáveis de Entrada (SQL): - NM_PESSOA_TOMADOR - Nome do cliente tomador - ID_PESSOA_TOMADOR - ID único do cliente - CD_RAMO - Código do ramo (filtro: 23, 28) - FL_DOCUMENTO_OFICIAL - Flag de documento oficial (filtro: 1) - DT_VIGENCIA_INI, DT_VIGENCIA_FIM - Período de vigência - VL_TARIFARIO - Valor tarifário da apólice

    Estrutura de Saída (Excel):

    Coluna Tipo Descrição
    NM_PESSOA_TOMADOR string Nome do cliente
    ID_PESSOA_TOMADOR int ID único cliente
    QDE_APOLICE_FIM_VIG int Apólices que venceram no período
    VL_TARIFARIO float Valor total tarifário
    QDE_NOVAS_APOLICES int Renovações efetivadas
    RENOVACAO string 'SIM' = renovado, 'NÃO' = não renovado
    ID string '131'
    NM_INDICADOR string 'Índice de Renovação RCO'

    Arquivo Gerado: 

    \\awsseg130006\ACTIO\ID - 131\ID 131 - Indice de Renovacao RCO - COMPETENCIA 202504 - gerado em 15-11-2024 h13 m42 s10.xlsx

    ID16 - Índice de Frequência de Sinistros (Danos Materiais)¶

    Objetivo: Medir a taxa de sinistralidade em Danos Materiais, indicando frequência de ocorrências.

    Fórmula: 

    Frequência de Sinistros DM = (Quantidade de Sinistros / Quantidade de Apólices) × 100

    Variáveis de Entrada (SQL): - QDE_APOLICES - Total de apólices ativas - QDE_SINISTROS - Total de sinistros ocorridos - CD_PRODUTO - Código do produto (produtos selecionados: 133, 124, 134, 163, 157, 169, 171, 184, 187) - DT_OCORRENCIA_SINISTRO - Data de ocorrência do sinistro

    Estrutura de Saída (Excel):

    Coluna Tipo Descrição
    CD_PRODUTO int Código do produto
    NM_PRODUTO string Nome do produto
    QDE_APOLICES int Quantidade de apólices do produto
    QDE_SINISTROS int Quantidade de sinistros ocorridos
    FREQUENCIA_SINISTROS float Taxa de frequência (%)
    ID string '16'
    NM_INDICADOR string 'Índice de Frequência de Sinistros Danos Materiais'

    ID17 - Recuperação de Sinistros (Acumulado por Data de Ocorrência)¶

    Objetivo: Medir a taxa de recuperação/ressarcimento em Danos Materiais acumulando por data de ocorrência do sinistro.

    Fórmula: 

    Recuperação = (Ressarcimento + Vendas Salvados) / Total de Sinistros Ocorridos

    Variáveis de Entrada (SQL): - CD_PRODUTO - Código do produto (filtro: 133, 124, 134, 163, 157, 169, 171, 184, 187) - DT_OCORRENCIA - Data de ocorrência do sinistro (eixo temporal) - VL_SINISTRO_OCORRIDO - Valor do sinistro - VL_RESSARCIMENTO - Valor ressarcido ao cliente - VL_VENDA_SALVADOS - Valor de venda de materiais salvados

    Estrutura de Saída (Excel):

    Coluna Tipo Descrição
    CD_PRODUTO int Código do produto
    DT_CORTE string Data de corte (YYYYMMDD)
    TOTAL_SINISTROS float Total de sinistros ocorridos
    TOTAL_RESSARCIDO float Total ressarcido + salvados
    TAXA_RECUPERACAO float Taxa em percentual (%)
    ID string '17'
    NM_INDICADOR string 'Recuperação Sinistros Danos Materiais'

    ⚠️ DIFERENÇA ID17 vs ID17m: - ID17: Agrupa por data de ocorrência do sinistro - ID17m (mensal): Agrupa por data de movimento (processamento)

    ID17m - Recuperação de Sinistros (Mensal por Data de Movimento)¶

    Objetivo: Medir a taxa de recuperação/ressarcimento em Danos Materiais agregando por data de movimento (processamento/processamento do sinistro).

    Fórmula: Mesma de ID17 

    Recuperação Mensal = (Ressarcimento + Vendas Salvados) / Total de Sinistros

    Diferença Principal com ID17: - Eixo temporal: Data de Movimento (em vez de Data de Ocorrência) - Permite análise de desempenho operacional (tempo de processamento)

    ID26 - Prazo Médio de Liquidação de Sinistros¶

    Objetivo: Medir o prazo médio (em dias) entre a ocorrência de um sinistro e sua liquidação final.

    Fórmula: 

    Prazo Médio = AVG(Data de Liquidação - Data de Ocorrência)

    Variáveis de Entrada (SQL): - DT_OCORRENCIA_SINISTRO - Data de ocorrência - DT_LIQUIDACAO - Data de liquidação final - CD_PRODUTO - Código do produto - VL_INDENIZACAO - Valor total da indenização

    Estrutura de Saída (Excel):

    Coluna Tipo Descrição
    CD_PRODUTO int Código do produto
    QDE_SINISTROS_LIQUIDADOS int Total de sinistros liquidados
    PRAZO_MEDIO_DIAS float Prazo médio em dias
    PRAZO_MINIMO_DIAS int Prazo mínimo observado
    PRAZO_MAXIMO_DIAS int Prazo máximo observado
    ID string '26'
    NM_INDICADOR string 'Prazo Médio Liquidação Sinistros'

    ID39 - Custo Médio de Sinistros (RCO)¶

    Objetivo: Medir o valor médio de indenização por sinistro em Responsabilidade Civil Operacional.

    Fórmula: 

    Custo Médio = Total de Custos Pagos / Quantidade de Sinistros

    Variáveis de Entrada (SQL): - QDE_SINISTROS - Quantidade de sinistros em RCO - VL_TOTAL_PAGO - Total pago (indenizações) - CD_RAMO - Código do ramo (filtro: RCO específico) - DT_PAGAMENTO - Data de pagamento

    Estrutura de Saída (Excel):

    Coluna Tipo Descrição
    CD_RAMO int Código do ramo
    NM_RAMO string Nome do ramo
    QDE_SINISTROS int Quantidade de sinistros
    VL_TOTAL_PAGO float Total pago em indenizações

    Como Criar um Novo Indicador¶

    Se precisar adicionar um novo indicador (ex: ID99 - Novo Indicador), siga estes passos:

    1️⃣ Definir Requisitos¶

    Documente: - Objetivo do indicador - Fórmula de cálculo - Filtros SQL necessários - Variáveis de entrada e saída

    2️⃣ Criar Classe Query (se necessário)¶

    Se a query é específica, crie uma nova classe Query em um arquivo auxiliar:

    # auxiliar_querys.py (criar se não existir)

from conexao_banco import Conexoes
from abc import ABC, abstractmethod

class MinhaNovaQuery(Query):
    def __init__(self, ambiente: str):
        super().__init__(ambiente)

    def query(self) -> tuple:
        """Retorna (sql_string, connection_object)"""
        sql = f"""
        SELECT 
            COLUNA1, COLUNA2, COLUNA3
        FROM 
            TABELA_ORIGEM
        WHERE 
            DATA >= '{{data_inicio}}'
            AND DATA <= '{{data_fim}}'
        """
        conexao = Conexoes(conexao=self.ambiente)
        return sql, conexao.abrir_conexao()

    3️⃣ Criar Arquivo do Indicador¶

    Crie arquivo padrão: ID99_Nome_Indicador.py

    # ID99_Nome_Indicador.py

import pandas as pd
from datetime import datetime
import APOIO
from auxiliar_querys import MinhaNovaQuery  # sua query

class GerarID99:
    def __init__(self, ambiente: str, data_execucao: datetime):
        self.ID_INDICADOR = '99'
        self.NOME_INDICADOR = 'Nome do Novo Indicador'
        self.DATA_COMP = data_execucao.strftime('%Y%m')
        self.ambiente = ambiente

    def gerar_indicador_99(self) -> pd.DataFrame:
        try:
            # 1. Criar pasta de saída
            PATH = APOIO.criar_pasta_indicador(self.ID_INDICADOR)

            # 2. Executar query
            query = MinhaNovaQuery(ambiente=self.ambiente)
            df = query.executar_query()

            # 3. Processar dados
            df.fillna(0, inplace=True)
            df.columns = [col.upper() for col in df.columns]
            df['ID'] = self.ID_INDICADOR
            df['NM_INDICADOR'] = self.NOME_INDICADOR

            # 4. Aplicar fórmula de cálculo (exemplo)
            df['MINHA_COLUNA_CALCULO'] = (df['COLUNA_A'] / df['COLUNA_B']) * 100

            # 5. Salvar arquivo Excel
            timestamp = APOIO.data_hora_geracao_arquivo()
            nome_arquivo = f"ID {self.ID_INDICADOR} - {self.NOME_INDICADOR} - COMPETENCIA {self.DATA_COMP} - gerado em {timestamp}.xlsx"
            arquivo_saida = f"{PATH}\\{nome_arquivo}"

            df.to_excel(arquivo_saida, sheet_name='INDICADOR', index=False)

            print(f"✓ ID99 gerado com sucesso: {arquivo_saida}")
            return df

        except Exception as e:
            print(f"✗ Erro ao gerar ID99: {str(e)}")
            raise

    Erro: "Falha na conexão SQL Server"¶

    Mensagem típica: 

    [ODBC Driver 18 for SQL Server] Named Pipe Provider: Could not open a connection to SQL Server

    Causas possíveis: 1. ❌ SQL Server não está acessível em 10.101.2.74:1433 2. ❌ Driver ODBC não instalado ou versão incorreta 3. ❌ Credenciais em .env estão incorretas

    Solução: 

    # 1. Validar conectividade (PowerShell)
Test-NetConnection -ComputerName 10.101.2.74 -Port 1433

# 2. Verificar ODBC driver instalado
Get-OdbcDriver | Select Name

# 3. Validar .env tem credenciais corretas
# Editar: .env
USER_PROD=seu_usuario
PASSWORD_PROD=sua_senha

# 4. Testar conexão com pyodbc
python -c "
import pyodbc
con = pyodbc.connect('Driver={ODBC Driver 18 for SQL Server};Server=10.101.2.74;Database=re_i4pro;UID=usuario;PWD=senha;Encrypt=no')
print('✓ Conexão OK')
"

    Erro: "Coluna não encontrada em SELECT"¶

    Mensagem típica: 

    pyodbc.ProgrammingError: ('42S22', "[42S22] [SQL Server]
Invalid column name 'NM_COLUNA_INEXISTENTE'. [SQL Server]")

    Causas possíveis: 1. ❌ Query SQL referencia coluna que não existe 2. ❌ Nome da coluna com tipografia errada 3. ❌ Alias de tabela incorreto

    Solução: 

    # 1. Testar Query SQL diretamente no SQL Server Management Studio
# Copiar SQL do arquivo .py e executar em SSMS

# 2. Validar nome das colunas:
SELECT COLUMN_NAME FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_NAME = 'MINHA_TABELA'

# 3. Se necessário, atualizar arquivo [ID X].py com coluna correta

    Erro: "Loop nunca termina / execução muito lenta"¶

    Causas possíveis: 1. ❌ Query SQL muito pesada (scans full table) 2. ❌ Volume de dados gigantesco no DataFrame 3. ❌ Loop infinito em processamento

    Solução: 

    # 1. Adicionar profiling de tempo
import time
t0 = time.time()
# ... seu código ...

print(f"Duração: {t1-t0:.2f}s")

# 4. Usar TQDM para progresso visual
