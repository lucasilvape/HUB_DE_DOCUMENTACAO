    Assistência Facil PP Residencial¶

    Pipeline ETL para extração e normalização de dados de seguros residenciais da plataforma FACIL PP (Fácil Proteção Patrimonial), processando certificados de seguros do PostgreSQL, transformando em formato padrão e transmitindo via FTPS (FTP-TLS) para servidor remoto seguro.

    ✨ Funcionalidades¶

    • Extração PostgreSQL: consultas à tabela financeiro.innoveo_financeiro_demais via psycopg2, filtrando parcelas ativas do produto 188

    • Processamento de certificados de seguros:

    • Filtragem por vigência: Remove certificados com vigência já expirada

    • Cálculo de data de assistência: Adiciona 32 dias à data inicial de vigência

    • Classificação de movimentação: Determina tipo de endosso (C = Contratação, A = Ativação, S = Suspensão) baseado em vigência e vencimento de parcelas

    • Detecção de vencimentos: Identifica parcelas com mais de 5 dias de atraso e marca como suspensas

    • Enriquecimento de metadados: Adiciona códigos de plataforma, companhia e tipo de movimento ao DataFrame final

    • Validação de integridade: Remove duplicatas e normaliza tipos de dados

    • Entrega via FTPS: Envio automático dos arquivos gerados com autenticação segura via FTP-TLS (porta 990) com certificado SSL/TLS customizável

    • Gerenciamento de credenciais: Variáveis de ambiente via .env, sem credenciais hardcoded

    • Suporte a processamento interativo: Notebook Jupyter incluído para debug e exploração de dados

    📋 Pré-requisitos¶

    Antes de começar, certifique-se de ter instalado em sua máquina:

    • Python 3.11+

    • Poetry 2.0+

    • Acesso ao banco de dados PostgreSQL (Innoveo / FACIL PP)

    • Acesso ao servidor FTPS de destino

    • Certificado SSL/TLS válido (Bundle .pem se customizado)

    🚀 Instalação¶

    1. Clone o repositório¶

    git clone <URL_DO_REPOSITORIO>
cd assistencia-facil-pp-residencial

    2. Instale as dependências¶

    poetry install

    3. Configure as variáveis de ambiente¶

    cp .env.example .env

    Edite o arquivo .env com as suas credenciais.

    ⚙️ Configuração¶

    Crie o arquivo .env na raiz do projeto com as seguintes variáveis:

    # ─── PostgreSQL (Innoveo / FACIL PP) ───────────────────────────
PG_HOST=
PG_DB=
PG_USER=
PG_PASSWORD=
PG_PORT=

# ─── FTPS (FTP-TLS) ────────────────────────────────────────────
FTP_HOST=
FTP_USUARIO=
FTP_PASSWORD=
FTP_CAFILE=         # (Opcional) Caminho para bundle de certificados SSL personalizados
                    # Se não definido, será usado o bundle padrão do certifi

    Nota: A porta FTPS é fixa em 990 (padrão para FTP-TLS implícito).

    Atenção: Nunca versione o arquivo .env. Ele já está (ou deve estar) listado no .gitignore.

    ▶️ Uso¶

    Executar o pipeline completo¶

    poetry run python facil_pp_residencial.py

    Este comando irá: 1. Conectar ao PostgreSQL 2. Extrair certificados de seguros residenciais com parcelas ativas 3. Aplicar transformações e validações 4. Gerar arquivo CSV com timestamp 5. Transmitir o arquivo via FTPS

    Executar via Notebook (análise interativa)¶

    jupyter notebook facil_pp_residencial_notebook.ipynb

    Útil para: - Explorar dados intermediários - Validar transformações passo a passo - Debugar queries e filtros

    Os arquivos gerados seguem o padrão de nomenclatura:

    Kovr_Facil_PP_Residencial_YYYYMMDD_HHMM.csv

    O arquivo é salvo em: 

    \\AWSSEG130006\I4proErp_Arquivos\Arquivos_Facil\

    📊 Fluxo de Processamento¶

    Ver fluxo completo do pipeline

    1. Conexão ao banco PostgreSQL¶

    Abre conexão com o PostgreSQL (Innoveo) via psycopg2 com as credenciais do .env.

    2. Extração dos dados¶

    Executa query na tabela financeiro.innoveo_financeiro_demais filtrando: - cd_produto = 188 (produto residencial FACIL PP) - Parcelas com status "ativa" - Certificados com data de vigência igual ou posterior à data de hoje

    3. Renomeação de colunas¶

    Mapeia nomes da tabela para padrão esperado: - bilhete → certificado_individual - dt_vigencia → data_inicio_vigencia - dt_fim_vigencia → data_fim_vigencia - etc.

    4. Filtragem por vigência¶

    Remove certificados cuja data_fim_vigencia seja anterior à data de hoje.

    5. Cálculo de data de assistência¶

    Calcula data_inicio_assistencia = data_inicio_vigencia + 32 dias (franquia padrão).

    6. Classificação de movimentação¶

    Determina tipo_movimento: - 'C' (Contratação): Se data_inicio_assistencia é posterior à data de hoje (futura) - 'A' (Ativação): Se data_inicio_assistencia é hoje ou anterior (já vigente) - 'S' (Suspensão): Se alguma parcela está vencida há mais de 5 dias

    7. Validação de vencimentos¶

    A função verifica_vencimento() valida: - Se alguma parcela do certificado venceu há mais de 5 dias - Se sim, marca como tipo_movimento = 'S' (suspensa)

    8. Limpeza de dados¶

    • Remove duplicatas via drop_duplicates()
    • Converte tipo de certificado para Int64 (nullable integer)
    • Reordena colunas em ordem padrão

    9. Enriquecimento de metadados¶

    Adiciona campos constantes: - codigo_plataforma: código da plataforma FACIL PP - codigo_cia: código da companhia - url_plataforma: URL de acesso à plataforma

    10. Exportação para CSV¶

    Salva o arquivo em codificação UTF-16 (com delimitador ;) no caminho de rede compartilhado:

    \\AWSSEG130006\I4proErp_Arquivos\Arquivos_Facil\Kovr_Facil_PP_Residencial_YYYYMMDD_HHMM.csv

    O arquivo é salvo com retry automático em caso de falha de acesso à rede (aguarda 1 segundo entre tentativas).

    11. Transmissão via FTPS (FTP-TLS)¶

    Abre conexão FTPS segura (porta 990) com certificado SSL/TLS via FTPTLS.openSession(): - Autenticação com usuário/senha do .env (FTP_USUARIO e FTP_PASSWORD) - Verifica certificado do servidor com TLS v1.2 mínimo - Usa bundle de certificados customizado se FTP_CAFILE definido; caso contrário, usa bundle padrão do certifi - Envia arquivo CSV para o servidor remoto (FTP_HOST) - Implementa prot_p() para ativar modo de dados criptografado

    🔧 Estrutura das funções principais¶

    facil_pp_residencial.py¶

    Fluxo principal:

    • Extração: Query PostgreSQL com filtragem (produto 188, parcelas ativas)

    • Renomeação de colunas: Mapeia nomes via dicionário columns_facil

    • Conversão de datas: pd.to_datetime() para vigência, cancelamento e vencimento

    • Filtragem por vigência: Remove certificados com dt_fim_vigencia_end < hoje

    • Cálculo de assistência: dt_inicio_assist = dt_ini_vigencia_end + 32 dias

    • Classificação: Tipo de movimento (C/A/S) baseado em datas e vencimentos

    • Validação de vencimentos: Marca como 'S' se parcela venceu há 5+ dias

    • Limpeza: Remove duplicatas mantendo primeira ocorrência

    • Enriquecimento: Adiciona metadados (tipo_endosso, cia, plataforma, cod_facil, etc.)

    • Reordenação: Reordena colunas conforme especificado

    • Exportação CSV: UTF-16, ponto-e-vírgula (;) como delimitador

    • Envio FTPS: Transmite arquivo via FTP-TLS à porta 990

    funcoes.py¶

    • verifica_vencimento(group) — Valida se QUALQUER parcela do grupo superou 5 dias de vencimento (retorna booleano)
    • class FTPTLS — Cliente FTPS customizado com suporte a certificados SSL/TLS (estende ftplib.FTP_TLS)
    • openSession(host, port, user, password, timeout) — Abre conexão FTPS segura e efetua login
    • connect() — Cria socket seguro com SSL/TLS v1.2+ (override customizado)
    • closeSession() — Encerra sessão com logging
    • makepasv() — Corrige host passivo para IP correto
    • Suporta certificados customizados via FTP_CAFILE ou bundle padrão do certifi

    conexoes.py¶

    • class conexao_banco — Gerencia credenciais PostgreSQL via variáveis de ambiente
    • HOST_BANCO, BANCO, USUARIO_BANCO, SENHA_BANCO, PORTA
    • class conexao_sftp — Carrega credenciais FTPS/SFTP do .env (estrutura reservada para futuro)
    • HOST, USUARIO, CHAVE, DIRETORIO, PORTA
    • Carregamento de variáveis via python-dotenv (requer arquivo .env na raiz)
