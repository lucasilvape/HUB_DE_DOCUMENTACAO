    ANALISE_PARCELAS_CONCILIACAO_PICPAY¶

    Automatiza a conciliação e análise de parcelas entre PicPay e Cyber, realizando importação de dados de múltiplas fontes (PostgreSQL, SQL Server, MongoDB), processamento ETL com validação, transformação conforme regras de negócio, e geração de relatórios de divergências com notificações condicionais.

    🔭 Visão geral do pipeline¶

    O orquestrador da execução é controlado pelos notebooks principais e scripts utilitários. O pipeline realiza:

    1. Validação de datas e períodos de processamento (diário, retroativo, completo).
    2. Leitura de dados de parcelas do PicPay em PostgreSQL.
    3. Leitura de dados de parcelas do Cyber em SQL Server.
    4. Processamento e transformação dos dados conforme regras de negócio.
    5. Reconciliação e identificação de divergências.
    6. Consolidação de resultados em estruturas de dados (DataFrames).
    7. Geração de relatórios e análises com exportação opcional para diferentes formatos.
    8. Notificações condicionais em caso de anomalias ou conclusão do processo.

    Observações importantes:

    • O pipeline é padrão em Jupyter Notebooks, permitindo exploração iterativa e validação de dados.
    • Suporta múltiplos ambientes: PRODUÇÃO, HOMOLOGAÇÃO e ANALYTICS para PostgreSQL.
    • Utiliza joblib para paralelização de operações quando necessário.
    • Variáveis de ambiente (.env) controlam credenciais e configurações conforme ambiente.
    • Estrutura modular em tools/ facilita reutilização de funções e conexões.

    🔄 Fluxo detalhado (end-to-end)¶

    Ver fluxo completo

    1. Entrada e validação inicial¶

    • Carregamento de parâmetros: datas de início/fim, ambiente, tipo de atualização.
    • Validação de conectividade a PostgreSQL, SQL Server e MongoDB via .env.
    • Cálculo de período de processamento (diário, retroativo ou completo).
    • Em caso de falha em conectividade: imprimir erro e encerrar o processo.

    2. Importação de dados PicPay¶

    • Consulta em financeiro.innoveo_financeiro_picpay_cyber (PostgreSQL/ANALYTICS).
    • Filtro por data_comp_pagamento e período especificado.
    • Transformação e limpeza de dados (conversão de tipos, tratamento de UUIDs, etc).
    • Consolidação em DataFrame estruturado.

    3. Importação de dados Cyber¶

    • Consulta em tabelas correspondentes do Cyber (SQL Server).
    • Aplicação dos mesmos filtros de período.
    • Transformação conforme esquema esperado.
    • Consolidação em DataFrame estruturado.

    4. Reconciliação e análise de divergências¶

    • Comparação entre PicPay e Cyber por chaves de conciliação (ex: ID da parcela, valor, data).
    • Identificação de registros únicos, divergências de valor e data.
    • Categorização de anomalias (faltante em PicPay, faltante em Cyber, diferença de valor, etc).

    5. Transformações de dados¶

    • Conversão de UUIDs (string ↔ bytes ↔ Binary UUID subtype 4 MongoDB).
    • Conversão de valores numéricos com vírgula para float.
    • Normalização de datas e formatos.
    • Tratamento de valores nulos e inválidos.

    6. Relatório e exportação¶

    • Impressão de estatísticas: quantidade de registros em cada fonte, divergências encontradas.
    • Exportação opcional para Excel com abas por tipo de divergência.
    • Visualizações exploratórias em caso de análise incremental.

    📁 Entradas, saídas e estrutura de pastas¶

    Entradas¶

    • Conexão PostgreSQL (bancos de origem: dados do PicPay).
    • Conexão SQL Server (dados de origem: dados do Cyber).
    • Conexão MongoDB (se necessário para enriquecimento de dados).
    • Arquivo .env com credenciais de múltiplos ambientes.

    Estrutura do repositório (principais itens)¶

    ANALISE_PARCELAS_CONCILIACAO_PICPAY/
├── picpay_cyber.ipynb                    (análise principal: conciliação PicPay ↔ Cyber)
├── script_diferencas_picpay_cyber.ipynb  (script de identificação de divergências)
├── README.md                             (este arquivo)
├── tools/
│   ├── __init__.py                       (inicializador do módulo)
│   ├── Conexao_exemplo.ipynb             (exemplos de conexão com diferentes BD)
│   ├── conexao_mongo.py                  (gerenciamento de conexões MongoDB)
│   ├── conexao_postgres.py               (gerenciamento de conexões PostgreSQL)
│   ├── conexao_sqlserver.py              (gerenciamento de conexões SQL Server)
│   ├── config_loader.py                  (carregamento de argumentos e configurações)
│   ├── funcoes_gerais.py                 (funções utilitárias: transformações, datas, UUIDs)
│   └── importacao_duck.py                (integração DuckDB, se aplicável)
└── .env_exemplo                          (modelo de arquivo de configuração)

    Saídas típicas¶

    • Relatórios em Excel com abas de divergências.
    • DataFrames com dados reconciliados em memória.
    • Logs de execução (console e opcional em arquivo).
    • Notificações por e-mail (em caso de erros críticos ou sucesso conforme configuração).

    🚀 Roadmap para Clonar e Executar o Projeto¶

    Ver passo a passo

    1. Pré-requisitos¶

    • Python 3.11 ou superior instalado
    • Git instalado
    • Jupyter Notebook ou JupyterLab instalado
    • Acesso ao PostgreSQL (ambientes: PRODUÇÃO, HOMOLOGAÇÃO, ANALYTICS)
    • Acesso a SQL Server (banco de Cyber)
    • Acesso a MongoDB (se necessário para enriquecimento)
    • Permissões para instalar dependências via pip

    2. Clonando o repositório¶

    Abra o terminal e execute: 

    git clone https://dev.azure.com/KovrSeguradora/BI/_git/ANALISE_PARCELAS_CONCILIACAO_PICPAY
cd ANALISE_PARCELAS_CONCILIACAO_PICPAY

    3. Instalando as dependências¶

    pip install 
# ou, se usando Poetry:
poetry install

    4. Configurando variáveis de ambiente¶

    • Renomeie o arquivo .env_exemplo para .env (ou crie um .env novo).

    • Preencha as variáveis conforme necessário:

    • Confirme que os bancos estão acessíveis conforme configurado.

    5. Executando os notebooks principais¶

    Análise completa de conciliação:¶

    jupyter notebook picpay_cyber.ipynb
    Siga as células do notebook para: - Conectar aos bancos de origem - Importar dados de PicPay e Cyber - Realizar análise de divergências - Gerar relatórios

    Script automatizado de diferenças:¶

    jupyter notebook script_diferencas_picpay_cyber.ipynb
    Para execução automatizada com parâmetros: 
    jupyter nbconvert --to notebook --execute script_diferencas_picpay_cyber.ipynb \
--ExecutePreprocessor.timeout=600 \
--output script_diferencas_picpay_cyber_output.ipynb

    6. Resultados esperados¶

    • Relatórios tabulares das divergências identificadas.
    • Exportações em Excel (se configurado).
    • Estatísticas resumidas impressas no console.
    • Logs de execução para auditoria.

    💡 Dica:
    Para testes locais, filtre períodos pequenos (ex.: uma semana) para validar conectividade e transformação de dados antes de rodar análises completas.

    🗄️ Banco de dados e conexões¶

    Leitura (Bancos de origem)¶

    Banco Tipo Tabelas Ambiente Função
    PostgreSQL ANALYTICS / PRODUÇÃO financeiro.innoveo_financeiro_picpay_cyber Dinâmico via .env Leitura de dados PicPay
    SQL Server Cyber dbo.parcelas_cyber (ou similar) Produção Leitura de dados Cyber
    MongoDB Analytics Coleções conforme necessário Dinâmico via .env Enriquecimento de dados (opcional)

    Configuração de conexões¶

    Consulte tools/conexao_*.py para detalhes: - conexao_postgres.py: Suporta múltiplos ambientes (função abrir_conexao_postgres()) - conexao_sqlserver.py: Conexão com SQL Server - conexao_mongo.py: Conexão com MongoDB

    Funções auxiliares¶

    • funcoes_gerais.py: Transformações de dados (UUID, conversão numérica, timestamps)
    • config_loader.py: Carregamento de argumentos de pipeline

    ✅ Critérios de sucesso e relatórios¶

    Validação da execução¶

    • Sucesso: Notebooks executam sem exceções, dados são importados de ambas as fontes, validações passam.
    • Divergências encontradas: Relatório detalhado é gerado com todas as anomalias identificadas.
    • Nenhuma divergência: Processos completam, relatório indica 100% conciliação.

    Estrutura de relatórios¶

    • Resumo geral: quantidade de registros em cada fonte, quantidade de divergências.
    • Detalhes por tipo de divergência:
    • Registros faltantes em PicPay
    • Registros faltantes em Cyber
    • Divergências de valor
    • Divergências de data
    • Outros
    • Recomendações de ação conforme tipo de divergência.

    ❓ Dúvidas frequentes e troubleshooting¶

    Ver respostas

    "Erro ao conectar em PostgreSQL"¶

    • Verifique se PG_HOST, PG_PORT, PG_DB, PG_USER, PG_PASSWORD estão corretos em .env.
    • Valide conectividade testando manualmente: 
      psql -h seu_host -U seu_usuario -d seu_banco
    • Confirme que o firewall permite acesso ao PostgreSQL.

    "Erro ao conectar em SQL Server"¶

    • Verifique credenciais SQLSERVER_* em .env.
    • Tente conectar via SQL Server Management Studio para validar.
    • Confirme que a máquina tem acesso de rede ao SQL Server.

    "Erro 'module not found' ao importar tools"¶

    • Certifique-se que você está na pasta raiz do repositório.
    • Confirme que tools/__init__.py existe.
    • Tente: 
      import sys
sys.path.append('.')
import tools.conexao_postgres as cp

    "DataFrame vazio após consulta"¶

    • Verifique se o período data_comp_pagamento especificado possui dados.
    • Valide a query em um cliente SQL diretamente (Ex: pgAdmin, DBeaver).
    • Confirme que está consultando a tabela correta.

    "Conversão de UUID falha"¶

    • Verifique o formato da UUID nos dados de origem (string com hífens, sem hífens, binária, etc).
    • Função codificar_text_para_bytes() em funcoes_gerais.py suporta múltiplos formatos.
    • Adicione logs para debugar o tipo de dado recebido.

    "Notebook demora demais ou trava"¶

    • Reduza o período de análise ou filtre por um subconjunto de dados.
    • Verifique se parallelização está habilitada conforme config_loader.py.
    • Monitore uso de memória (RAM) durante execução.

    "Relatório em Excel não é gerado"¶

    • Confirme que openpyxl está instalado: 
      pip install openpyxl
    • Valide permissões de escrita na pasta onde o arquivo será salvo.

    "E-mail de notificação não é enviado"¶

    • Verifique configuração SMTP em .env (se implementado).
    • Confirme que credenciais de e-mail estão corretas.
    • Verifique se a função de envio está sendo chamada no notebook.
