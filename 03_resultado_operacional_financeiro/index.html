    Resultado Operacional Financeiro¶

    Automatiza o fechamento operacional e financeiro mensal, integrando múltiplas fontes de dados do i4Pro, processando prêmios, sinistros e PPNG com validação, reconciliação, inserção em DW e notificações por e-mail somente quando necessário.

    🔭 Visão geral do pipeline¶

    O orquestrador da execução é o arquivo 00_importar_fechamento.py. Ele realiza:

    1. Validação de datas e períodos de processamento.
    2. Importação de prêmios/cancelamentos para múltiplos ramos.
    3. Importação de sinistros para múltiplos ramos.
    4. Importação de PPNG (Provisão de Prêmios Não Ganhos).
    5. Processamento e transformação dos dados conforme regras de negócio.
    6. Inserção das atualizações no DW (kovr_dw_bi).
    7. Inserção de logs de execução no banco de logs.
    8. Relatório final e notificações condicionais.

    Observações importantes¶

    • O pipeline processa de forma paralela usando Parallel e delayed do joblib para otimizar performance.
    • E‑mail: enviar apenas em caso de erro ou com verificação explícita de sucesso quando houver inserções; silenciar quando o processo correr sem novos registros.
    • Múltiplos bancos de origem: re_i4pro, vida_i4pro, vida_seg_i4pro são consultados conforme configurado.
    • Variáveis de ambiente (.env) controlam credenciais e URLs conforme ambiente (produção ou homologação).

    Configurações usadas no 00_importar_fechamento.py (padrão atual)¶

    • Listas de e‑mail definidas no código:
    • DESTINATARIOS: recebe resumos de execução (informativo)
    • Período processado: definido por start (data mínima corporativa) e end (último dia do mês vigente)

    🔄 Fluxo detalhado (end-to-end)¶

    Ver fluxo completo

    1. Entrada e validação inicial

      • Leitura de parâmetros: datas start/end, lista de bancos, ambientes i4 e BI, destinatários.
      • Validação de conectividade e credenciais via .env.
      • Cálculo de intervalo de datas mensal usando pd.date_range.
      • Em caso de falha em conectividade: imprimir erro e enviar e-mail; encerrar o processo.

    2. Processamento de Prêmios

      • EfetuarImportacaoPremios: leitura de dados de emissão/cancelamento no i4 (producao_i4).
      • Transformação e preparação conforme esquema esperado.
      • Inserção em kovr_dw_bi e log em kovr_logs_processos.

    3. Processamento de Sinistros

      • EfetuarImportacaoSinistros: query de sinistros ativos/inativos do i4.
      • Validação de status e preparação de dados.
      • Inserção em DW.

    4. Processamento de PPNG

      • EfetuarImportacaoPPNG: cálculo de provisões de prêmios não ganhos por cobertura/ramo.
      • Consolidação de valores mensais.
      • Inserção em DW.

    5. Relatório

      • Impressão de estatísticas gerais: quantidade de registros processados, inseridos, erros.
      • Envio de e-mail conforme configuração (sucesso informativo ou erro crítico).

    📁 Entradas, saídas e estrutura de pastas¶

    Entradas

    • Conexões SQL Server (bancos: re_i4pro, vida_i4pro, vida_seg_i4pro).
    • Arquivo .env com credenciais e URLs de ambiente.

    🚀 Roadmap para Clonar e Executar o Projeto¶

    Ver passo a passo

    1. Pré-requisitos¶

    • Python 3.11 ou superior instalado
    • Git instalado
    • Poetry instalado
    • Acesso ao SQL Server (bancos re_i4pro, vida_i4pro, vida_seg_i4pro, kovr_dw_bi, kovr_logs_processos)
    • Permissões para instalar dependências

    2. Clonando o repositório¶

    Abra o terminal e execute:

    git clone https://dev.azure.com/KovrSeguradora/BI/_git/03_RESULTA_OPERACIONAL_FINANCEIRO
cd 03_RESULTA_OPERACIONAL_FINANCEIRO

    3. Instalando as dependências e ativando o ambiente¶

    poetry install
poetry env activate
# (caso você feche o terminal e precise ativar o ambiente virtual novamente)

    4. Configurando variáveis de ambiente¶

    • Renomeie o arquivo .env_exemplo para .env (ou crie um .env novo).
    • Preencha as variáveis conforme necessário:
    USER_PROD=seu_usuario_sql_server_producao
PASSWORD_PROD=sua_senha_sql_server
USER_DW_BI=seu_usuario_dw_bi
PASSWORD_DW_BI=sua_senha_dw_bi
USER_EMAIL_BI=seu_email
PASSWORD_EMAIL_BI=sua_senha_email
URL_PROD=https://api.producao.i4pro.com.br
URL_HOMOLOG=https://api.homolog.i4pro.com.br
    • Confirme que os bancos listados estão acessíveis conforme configurado no script.
    • Para testes locais, ajuste start e end em 00_importar_fechamento.py para períodos pequenos.

    5. Executando o pipeline¶

    Configurações usadas no 00_importar_fechamento.py:

    • LISTA_BANCOS_DB: bancos a processar ['re_i4pro','vida_i4pro','vida_seg_i4pro']
    • Ambientes: AMBIENTE_i4='producao_i4' e AMBIENTE_BI='dw_bi'
    • Destinatários de e-mail definidos no código
    • Período: start='2026-01-01' e end='2026-01-31' (ajustável)
    python 00_importar_fechamento.py

    6. Resultados¶

    • O processo imprime o relatório final no terminal (resumo de registros processados).
    • Inserções são feitas diretamente nos bancos kovr_dw_bi e kovr_logs_processos.
    • E-mails são enviados apenas em caso de erro ou conforme configuração de sucesso.

    💡 Dica: Se for rodar em ambiente de produção (ex: Jenkins), garanta que o .env esteja presente no workspace com as variáveis corretamente preenchidas.

    📧 Regras de notificação por e-mail¶

    • Erros críticos: enviar e-mail com detalhes (falha na conexão, exceções nas queries/transformações, erro na inserção em banco, etc.).
    • Sucesso com inserções: imprimir resumo e opcionalmente enviar e-mail informativo (quantidade inserida por tipo: prêmios, sinistros, PPNG).
    • Execução sem novos registros: não enviar e-mail (execução silenciosa, apenas logs locais).

    🗄️ Banco de dados e conexões¶

    • Leitura (Bancos de origem): conexão producao_i4 (utilizada nas queries de consulta de prêmios, sinistros, PPNG).
    • Escrita (DW): conexão dw_bi (utilizada para inserir em kovr_dw_bi.dbo.*).
    • Logs: conexão logs_bi (utilizada para registrar execução em kovr_logs_processos.dbo.log_execucao).
    • Tabelas principais de destino:
    • dim_emissao_cancelamento (prêmios)
    • dim_sinistros (sinistros)
    • dim_ppng (provisões)
    • Consulte conexao_banco.py e autenticacao.py para a configuração dos aliases de conexão.

    ✅ Critérios de sucesso e relatórios¶

    • Inserção em banco:
    • Sucesso: funções de inserção retornam True e são impressos os totais; e-mail opcional de sucesso.
    • Falha: retorna False e envia e-mail de erro com detalhes.
    • Vazio: o main imprime "Nenhum registro novo para inserir." e não chama a função.
    • Relatórios:
    • Sempre imprimir estatísticas gerais da execução (período, bancos, totais processados).
    • Imprimir distribuição por tipo (prêmios, sinistros, PPNG) apenas quando houver registros (> 0).

    ❓ Dúvidas frequentes e troubleshooting¶

    Ver respostas

    • "O pipeline roda, mas não há inserções no DW."

      • Verifique se o período definido (start/end) possui dados nos bancos de origem.
      • Confirme conectividade aos bancos re_i4pro, vida_i4pro, vida_seg_i4pro via conexao_banco.py.
      • Valide que as queries estão retornando registros (ative prints de debug).

    • "Erro de credenciais ao iniciar."

      • Revise .env e confirme que as variáveis USER_PROD, PASSWORD_PROD, USER_DW_BI, PASSWORD_DW_BI estão preenchidas.
      • Teste conectividade manualmente via SQL Server Management Studio.

    • "E-mail não está sendo enviado."

      • Verifique USER_EMAIL_BI e PASSWORD_EMAIL_BI em .env.
      • Confirme que a função está sendo chamada (verifique enviar_email.py).
      • Valide credenciais SMTP ou integração com servidor de e-mail da empresa.

    • "Parallelização está lenta."

      • Reduza N_JOBS em *_importar.py (ex.: de 10 para 4) segundo a capacidade da máquina/rede.
      • Monitore consumo de conexões SQL Server.
